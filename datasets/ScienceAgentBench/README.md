# ScienceAgentBench Dataset Overview

  ScienceAgentBench is a comprehensive benchmark designed to evaluate language agents for data-driven scientific discovery. The dataset contains 102 diverse tasks across multiple scientific domains, with a focus on
  computational analysis, machine learning, and data visualization.

  Dataset Structure

  The dataset is organized in a CSV file (ScienceAgentBench.csv) with each row representing a task instance. The main structure includes:

   - ID: Instance number (1-102)
   - Domain: Scientific field (Computational Chemistry, Bioinformatics, Geographical Information Science, Psychology/Cognitive Science)
   - Subtasks: Categories of tasks (Feature Engineering, Machine Learning, Deep Learning, Statistical Analysis, Data Visualization, etc.)
   - GitHub Source: Original repository the task was adapted from
   - Task Instructions: Natural language description of what needs to be accomplished
   - Domain Knowledge: Explanations of scientific concepts, formulas, and methodologies
   - Dataset Preview: Directory structure and sample content
   - Source Files: Input data and reference notebooks
   - Gold Program Names: Reference implementations of the tasks
   - Output File Names: Expected output locations
   - Evaluation Programs: Automated assessment scripts

  Key Domains and Tasks

  Computational Chemistry (31 tasks)
   - Molecular modeling (toxicity prediction, band gap prediction)
   - Protein-ligand interactions and drug-target binding
   - Materials science (crystal structures, electronic properties)
   - Molecular visualization and analysis

  Bioinformatics (38 tasks) 
   - Gene expression analysis and single-cell RNA sequencing
   - Drug discovery and ADMET prediction
   - Molecular property prediction
   - Biological pathway analysis

  Geographical Information Science (21 tasks)
   - Spatial analysis and mapping
   - Environmental monitoring and change detection
   - Geospatial modeling and visualization
   - Climate and oceanographic data analysis

  Psychology/Cognitive Science (12 tasks)
   - Biosignal processing (ECG, EEG, EOG)
   - Behavioral analysis and questionnaire data
   - Neural signal processing
   - Cognitive modeling

  Evaluation Framework

  The evaluation system consists of:

   1. Gold Programs: Reference implementations in Python
   2. Evaluation Programs: Scripts that automatically assess functionality and correctness
   3. Scoring Rubrics: Detailed point-based evaluation criteria for each task
   4. Metrics: Valid Execution Rate, Success Rate, CodeBERTScore, and API Cost

 EVALUATION
While it is a preferable feature, the open-endedness of tasks in our benchmark introduces a crucial evaluation challenge. Specifically, our evaluation strategy has to accommodate diverse setup
requirements of programs generated by different agents. To address this challenge, we implement
a pipeline to set up a conda environment flexibly for any program. Before evaluation, the conda
environment is initialized with seven basic Python packages: numpy, pandas, matplotlib, pytorch,
tensorflow, rdkit, and tf keras. To evaluate each program, we first use pipreqs1
to analyze it and generate a file listing all packages used. Then, according to the file, we use pip-tools2
and handcrafted rules to update the conda environment and properly configure the packages. We execute each
program in the customized environment and calculate the evaluation metrics.
Program Evaluation. We comprehensively evaluate each generated program with four metrics. (1)
Valid Execution Rate (VER) checks if the program can execute without errors and save its output
with the correct file name. (2) Success Rate (SR) examines whether a program output meets the
success criteria for each task goal (Table 1), such as test set performance, prediction-answer matches,
and visualization quality. To automatically check these criteria, we implement them as evaluation
programs for each task during annotation. By nature, SR is conditioned on valid execution: If a
program has execution errors or does not save its output correctly, its SR will be 0. Both VER and
SR are binary metrics. (3) CodeBERTScore (CBS) measures how closely the generated program
resembles the annotated one with contextual embeddings and calculates the F1 metric for matched
token embeddings (Zhou et al., 2023). If SR = 1 for a program, we change its CBS to 1.0 as well to
reflect task success. (4) API Cost (Cost) calculates the average cost (in USD) to complete one task
in our benchmark, since it is important for language agents to control their cost and optimize their
design for better practical utility (Kapoor et al., 2024).

  Data Formats

  The dataset includes various scientific data formats:
   - CSV/TXT for tabular data
   - SDF for molecular structures
   - GeoJSON for geospatial data
   - H5AD for single-cell data
   - NetCDF for climate data
   - PDB for protein structures
   - Various image formats for molecular visualization

  Assessment Methodology

  Each task evaluates agents on their ability to:
   - Understand scientific concepts and methodologies
   - Process complex scientific data formats
   - Apply appropriate computational tools and libraries
   - Generate functional, correct code
   - Produce expected outputs in specified formats

  This benchmark represents a comprehensive evaluation framework for language models in scientific computing, requiring deep understanding of scientific concepts, domain-specific libraries, and complex data
  processing tasks.
